---

### 📖 README 规范
生成的 README.md 必须包含：
- **项目简介**: 一句话描述项目价值。
- **技术栈**: 使用的核心技术和版本。
- **快速开始**: 复制粘贴即可运行的安装与启动命令。
- **环境变量**: 必须配置的变量列表及示例。

### 💬 代码注释
- **JSDoc/Docstrings**: 公共函数和类必须包含标准文档注释。
- **TODO 标记**: 对于临时方案或未完成的逻辑，必须使用 `// TODO: [描述]` 标记。
- **复杂逻辑**: 如果算法复杂，必须在代码上方用自然语言解释算法思路。

### 🚫 禁止事项
- 不要生成过时的文档。
- 不要解释显而易见的代码 (如 `i++ // i 加 1`)。
