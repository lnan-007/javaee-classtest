### 🧪 测试原则
- **AAA 模式**: 所有测试必须遵循 Arrange (准备), Act (执行), Assert (断言) 结构，并用注释分隔。
- **独立性**: 测试用例之间不得有依赖，每个测试必须能独立运行。
- **命名规范**: 测试函数名必须描述行为，格式为 `should_预期结果_当_条件` (如 `should_return_404_when_user_not_found`)。

### 🛠️ 模拟与桩
- **Mock 外部依赖**: 严禁在单元测试中调用真实的 API 或数据库。必须使用 Mock 或 Stub 模拟外部服务。
- **不要测试实现细节**: 只测试公共接口的行为，不要断言私有方法是否被调用。

### 📊 覆盖率要求
- 核心业务逻辑覆盖率必须达到 90% 以上。
- 必须包含边界值测试 (空值、极大值、负数)。
- 必须包含异常路径测试 (错误输入、网络失败)。
